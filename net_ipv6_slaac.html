SLAAC - Stateless Address Autoconfiguration


Stateless Address Autoconfiguration (SLAAC) ist ein Verfahren zur automatischen und zustandslosen Konfiguration von IPv6-Adressen an einem Netzwerk-Interface. Im Gegensatz zur klassischen IP-Autokonfiguration bei IPv4 wird bei SLAAC die IPv6-Adresse nicht zentral vergeben oder gespeichert. Stattdessen erzeugt sich der Host seine IPv6-Adresse selbst, indem er zusätzliche Informationen verwendet. SLAAC ist eine Weiterentwicklung der Autokonfigurationstechniken von IPv4, bei der IPv6-Router eine aktive Rolle spielen.

Es gibt zwei Haupttypen von IPv6-Adressen, die mit SLAAC generiert werden: Globale IPv6-Adressen (Global Scope) und link-lokale IPv6-Adressen (Local Scope). Durch die Verwendung von Stateless Address Autoconfiguration kann ein IPv6-Host sowohl eine link-lokale Adresse als auch eine globale Adresse erzeugen. Dadurch bietet IPv6 den gleichen Komfort wie der Betrieb eines einfachen DHCP-Servers.

Das Hauptziel von SLAAC besteht darin, dass ein Host mindestens eine link-lokale IPv6-Adresse erhält, um eine Verbindung innerhalb des lokalen Netzwerks herzustellen. Anschließend kann der Host mithilfe von SLAAC eine globale IPv6-Adresse erzeugen, um Verbindungen zum Internet herzustellen.

Durch die automatische Konfiguration von IPv6-Adressen mittels SLAAC wird die Netzwerkkonfiguration vereinfacht und Hosts können sich effizient selbst adressieren, ohne dass eine manuelle Konfiguration oder ein DHCP-Server erforderlich ist. Dadurch wird die Skalierbarkeit und Flexibilität von IPv6 verbessert und die Inbetriebnahme von Geräten in einem Netzwerk erleichtert.

Wie funktioniert SLAAC?

1. Router Advertisement (RA): IPv6-Router senden periodisch sogenannte Router Advertisement-Nachrichten an das lokale Netzwerk. Diese Nachrichten enthalten wichtige Informationen für die Hosts im Netzwerk, einschließlich des Präfixes, das für die Bildung der IPv6-Adresse verwendet wird.

2. Network Prefix: Der Router gibt den Hosts im Netzwerk ein Präfix bekannt, das für die Bildung der IPv6-Adresse verwendet wird. Das Präfix definiert den Netzwerkanteil der Adresse.

3. Interface Identifier: Der Host generiert den Interface Identifier, der den individuellen Teil der IPv6-Adresse darstellt. Hierfür gibt es verschiedene Möglichkeiten:

- EUI-64: Der Interface Identifier kann aus der 48-Bit-MAC-Adresse des Netzwerkadapters abgeleitet werden. Dabei werden die ersten 24 Bits der MAC-Adresse umgedreht und ein fester Wert (0xFFFE) eingefügt.

- Random: Alternativ kann der Interface Identifier auch zufällig generiert werden, ohne Bezug zur MAC-Adresse.

4. Zusammensetzung der Adresse: Der Host kombiniert das empfangene Präfix mit dem generierten Interface Identifier, um die vollständige IPv6-Adresse zu bilden.

5. DAD (Duplicate Address Detection): Bevor der Host die konfigurierte IPv6-Adresse verwendet, führt er eine Duplikaterkennung durch, um sicherzustellen, dass keine andere Maschine im Netzwerk bereits die gleiche Adresse verwendet.

6. Adresskonfiguration: Nach erfolgreicher Duplikaterkennung ist die IPv6-Adresse des Hosts konfiguriert und kann für die Netzwerkkommunikation verwendet werden.

Es ist wichtig zu beachten, dass SLAAC in Kombination mit Router Advertisements (RA) und Neighbor Discovery Protocol (NDP) arbeitet, um die Autoconfiguration und Kommunikation im IPv6-Netzwerk zu ermöglichen. Router Advertisement-Nachrichten enthalten neben dem Präfix auch andere wichtige Informationen wie die Router-Präferenz und die Router Lifetime, die angeben, wie lange ein Präfix gültig ist.
